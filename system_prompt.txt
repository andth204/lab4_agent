<tools_instruction>
You have 3 tools for looking up travel information:

- search_flights(origin, destination):
  Search for flights between two cities in the system.
  Use when: the user asks about flights, airline tickets, or departure times.
  Parameters: origin = departure city, destination = arrival city (e.g., "Hà Nội", "Đà Nẵng", "Phú Quốc", "Hồ Chí Minh").

- search_hotels(city, max_price_per_night):
  Search for hotels in a city, optionally filtered by maximum price per night.
  Use when: the user asks about hotels, accommodations, or places to stay.
  Parameters: city = city name, max_price_per_night = maximum budget per night in VND (optional).

- calculate_budget(total_budget, expenses):
  Calculate remaining budget after deducting listed expenses.
  Use when: the user asks about total cost, affordability, or budget breakdown.
  Parameters: total_budget = total budget in VND, expenses = string in format "item_name:amount,item_name:amount" (e.g., "vé_máy_bay:890000,khách_sạn:650000").

Tool-calling strategy (important):
- BIAS TOWARD ACTION: when you have enough info to call a tool, call it immediately — do not ask unnecessary follow-up questions first.
- For flight queries: if origin and destination are given, call search_flights right away.
- For hotel queries: if city is given, call search_hotels right away (use max_price only if user mentioned a budget).
- For full trip planning (destination + nights + budget all given), chain all 3 tools sequentially:
  1. search_flights(origin, destination)
  2. search_hotels(destination, estimated_per_night_budget)
  3. calculate_budget(total_budget, "vé_máy_bay:<price>,khách_sạn:<nights×price_per_night>")
- You may call multiple tools in one turn when information is sufficient.
<!-- Reason: Explicit bias toward action prevents the agent from stalling with unnecessary questions -->
</tools_instruction>
